
## Creating Log Listener Service

- By default the python script will be manual so the below steps are required to make it run at startup so that we get log updates in real time without manually running the script.
- To create the log listener service follow the below steps.

### PROCESS

1. **FIND PYTHON PATH** : `which python3`
2. **CREATE SERVICE FILE** :
	- Enter the command `sudo nano /etc/systemd/system/ai-log-listener.service` and paste the below details inside that file.
	- `<working directory path>` : Provide the working directory location where the code file is using `pwd` command.
	- `<python filepath>` : Inside here provide your python3 file path.
	- `<log listener.py path>` : Provide the full path to your `log_listener.py`. If you followed the previous cowrie service setup then the `log_listener.py` should be inside the `cowrie_ai` directory.

```
[Unit]
Description=AI Log Listener for Cowrie
After=docker.service
Requires=docker.service

[Service]
User=kali
WorkingDirectory=<working directory path>
ExecStart=<python filepath> <log listener.py path>/log_listener.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

3. **RELOAD SYSTEMD** : `sudo systemctl daemon-reload`
4. **ENABLE SERVICE AT BOOT** : `sudo systemctl enable ai-log-listener.service`
5. **START THE SERVICE** : `sudo systemctl start ai-log-listener.service`
6. **TESTING** : Check the status using `sudo systemctl status ai-log-listener.service`
