

## Creating a Cowrie Service

- By default we need to start our cowrie through the docker.
- To make this process automated we can create a `cowrie.service` so that on startup of our kali VM the cowrie gets started.
- To do this follow the below steps carefully.

### Process

1. **GET REAL PATH** : We have our cowrie in a directory named `/cowrie/docker`. Find the real path to this directory using `pwd` command first. For example mine is `/home/kali/cowrie/docker`
2. **CREATE SYSTEMD SERVICE** : `sudo nano /etc/systemd/system/cowrie.service` . Then paste the below stuff in that file and put your real directory path in the `<real directory path>` section. Then save and exit.

```
[Unit]
Description=Cowrie Honeypot Docker Compose
Requires=docker.service
After=docker.service

[Service]
Type=oneshot
RemainAfterExit=yes
WorkingDirectory=<real directory path>/cowrie/docker
ExecStart=/usr/bin/docker compose up -d
ExecStop=/usr/bin/docker compose down
TimeoutStartSec=0

[Install]
WantedBy=multi-user.target
```

3. **RELOAD SYSTEMD** : `sudo systemctl daemon-reload`
4. **ENABLE SERVICE AT BOOT** : `sudo systemctl enable cowrie.service`
5. **TESTING** : Restart the system using `reboot` command and then enter the `docker ps` command. You should see something like `docker-cowrie-1   Up XX seconds`
